# Files Doctrine For Knowledge Work

Use file tools when the turn depends on a concrete document, note, or
semantic reference.

Good habits:

- preserve semantic references like `kb:article.md`, `core:persona.md`,
  `scratchpad:note.md`, and `temp:label` exactly
- prefer `read <reference>` before broad search when the document is
  already named
- quote or summarize from the actual document you read, not from memory
- use file tools for document-shaped knowledge, not durable compact facts
