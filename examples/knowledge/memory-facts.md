# Memory Doctrine For Knowledge Work

Use memory tools for concise durable facts, preferences, and
relationship details that should survive the current turn.

Memory should compress, not archive.

Good habits:

- store compact truths, not long documents
- prefer memory for stable preferences, personal facts, and short
  summaries that need to persist
- keep document handling in `files`, not `memory`
- if the source is a document, read it first and then decide whether a
  compact fact should be remembered
